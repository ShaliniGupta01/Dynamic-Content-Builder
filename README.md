# 🚀 Dynamic Content Builder

A modern, interactive web application that allows users to **dynamically build and customize a personal content page** using draggable and configurable components.

---

## 📌 Project Overview

The Dynamic Content Builder is a **single-page React application** where users can:

* Add different types of content blocks (Text, Image, Header)
* Drag & reorder blocks
* Edit content in real-time
* Save layouts and reload them anytime
* Manage multiple saved layouts with rename & delete options

---

## ✨ Features

### 🧩 Content Blocks

* Text Block (editable textarea)
* Image Block (URL-based rendering)
* Header Block (customizable heading)

---

### 🎯 Drag & Drop

* Smooth drag-and-drop functionality using **@dnd-kit**
* Reorder blocks easily

---

### ✏️ Editing & Customization

* Real-time editing of all blocks
* Update content instantly
* Rename saved layouts

---

### 💾 Layout Management

* Save layouts with a single click
* Load any saved layout
* Delete layouts
* Rename layouts dynamically

---

### 🔄 Persistence

* All layouts are stored in **localStorage**
* Data persists even after page refresh

---

### 🎨 UI/UX

* Clean and responsive UI using **Tailwind CSS**
* Interactive hover effects
* User-friendly layout structure

---

## 🛠️ Tech Stack

* ⚛️ React.js (Vite)
* 🎨 Tailwind CSS
* 🧠 JavaScript (ES6+)
* 📦 @dnd-kit (Drag & Drop)
* 💾 LocalStorage (Persistence)

## ⚙️ Installation & Setup

1. Clone the repository:

```bash
git clone https://github.com/your-username/content-builder.git
```

2. Navigate to project folder:

```bash
cd content-builder
```

3. Install dependencies:

```bash
npm install
```

4. Run the project:

```bash
npm run dev
```

## 🧠 Key Architectural Decisions

* **Component-Based Architecture** for scalability and maintainability
* **State Management using React Hooks**
* **Immutable state updates** for reliable UI rendering
* **LocalStorage integration** for persistence
* **Reusable components** for better code organization

---

## ⚡ Challenges & Solutions

| Challenge              | Solution                          |
| ---------------------- | --------------------------------- |
| Drag & drop complexity | Used @dnd-kit for better control  |
| State synchronization  | Implemented immutable updates     |
| Data persistence       | Integrated localStorage           |
| UI responsiveness      | Used Tailwind CSS utility classes |
